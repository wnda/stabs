stabs.js

What?

stabs.js is about 436 bytes of JavaScript for creating tabbed sections of your webpage.

It's not something I'm proud of. It's not friendly, it doesn't have some clean API and it's not a module.

You stick it at the bottom of the <body> and name your tabbed elements according to the lib's requirements.

Why?

Bootstrap and other libraries/frameworks tend to require jQuery. Not surprising. My 50 lines of vanilla would probably be about 8 lines of jQuery.

However, it is 2016 and ES5/ES6 are gaining traction, and the ES3 that is actually supported in the real world is now largely standardised even in IE.

There are of course some vanilla JS alternatives, but they all navigate to hash fragments in the URL bar, which is a feature I explicitly didn't want.

I wanted something lean, robust and amenable to nesting because, though I do not tend to nest things (I'm not a fan of complexity) I do believe it is important to be prepared for the day when nesting will be required.

Thus, stabs.js was born.

How?

stabs.js uses only vanilla JavaScript primarily relying on for loops and classList.

This small lib loops through everything in the DOM with a class of “tabcontainer” and then adds event listeners to everything that is a “tab” in that tabcontainer.

These event listeners wait for a tab to be clicked and then, if the tab is not active, it and its corresponding tabpane are set to active and all other tabs/tabpanes have their active status removed.

This might seem like cheating, but the approach could easily be adapted to manipulate styles directly. However, appending and removing classes is ultimately more efficient, and also enables the exact way tabs are displayed/hidden to be very flexible to the CSS-savvy dev.

This is ultimately the simplest way to work with tabs.

Using it?

Create an element with a class of “tabcontainer”.

Within that, create elements with a class of “tab”.

For each “tab”, make sure you create a corresponding element with a class of “tabpane”—all within the “tabcontainer”.

Include the library at the bottom of your <body> and it'll do its job.

More info?

IE9, or IE8 if you polyfill classList.

I experimented with className but the code was horribly ugly and inefficient.
